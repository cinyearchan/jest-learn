### BDD

1. 先写代码再写测试
2. 一般结合集成测试使用，是黑盒测试
3. 测试重点在UI(DOM)
4. 安全感高
5. 速度慢

### benefits

1. 代码组织、可维护性 ↑
2. bug(尤其是回归测试中的) ↓
3. 修改工程质量差的项目，安全 ↑
4. 项目具备潜在的文档特性
5. 拓展前端知识面

#### tips

1. BDD TDD
2. 集成测试 单元测试
3. 测试与业务的解耦
4. 代码测试覆盖率并不代表一定靠谱
5. 功能测试及 UI 测试
